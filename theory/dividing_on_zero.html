Dividing In Lagrange basis when one of the points is zero - Generalised

Problem

We restate the problem, however we do not assume that the elements in the domain are roots of unity.

We have g(X)f(X)​=X−xm​f(X)​=∑i=0d−1​fi​X−xm​Li​(X)​

Lagrange polynomial

We briefly restate the formula for a lagrange polynomial:

Li​(X)=j=i,j=0∏​xi​−xj​X−xj​​

The i'th lagrange polynomial evaluated at xi​ is 1 and 0 everywhere else on the domain

First form of the barycentric interpolation formula

We introduce the polynomial A(X)=(X−x0​)(X−x1​)...(X−xn​).

We also introduce the derivative of A′(X)=∑j=0d−1​∏i=j​(X−xi​) .

You can derive this yourself by generalising the product rule: https://en.wikipedia.org/wiki/Product_rule#Product_of_more_than_two_factors

In general this derivative does not have a succint/sparse form. We do have a succint form if the domain is the roots of unity!

Now note that A′(xj​)=∏i=0,i=j​(xj​−xi​)

If we plug in xk​ into A′(X) all the terms with X−xk​ will vanish, this is why the sum disappears into a single product.

We can use A and A′ to re-define our lagrange polynomial as :

Li​(X)=A′(xi​)(X−xi​)A(X)​

Looking at the original lagrange formula, A′(xi​) is the denominator and X−xi​A(X)​ is the numerator.

The first barycentric form for a polynomial f(X) can now be defined as :

f(X)=i=0∑d−1​A′(xi​)(X−xi​)A(X)​fi​

Remarks

  • A(X) is not dependent on the values of fi​ and so can be brought out of the summation.
  • A′(X) is only dependent on the domain, so it can be precomputed, along with A(X)

Re-defining the quotient

Note that our original problem was that:

i=0∑d−1​fi​X−xm​Li​(X)​

Had a X−xm​ term in the denominator. We will use the first form as a way to get rid of this.

First we rewrite X−xm​Li​(X)​ using the first form:

X−xm​Li​(X)​=A′(xi​)(X−xi​)(X−xm​)A(X)​

We then note that:

A(X)=Lm​(X)⋅A′(xm​)⋅(X−xm​)

I just re-arranged the formula for the first form to equal A(X) for Lm​(X)

We can hence plug this into our previous equation:

X−xm​Li​(X)​=A′(xi​)(X−xi​)(X−xm​)Lm​(X)⋅A′(xm​)⋅(X−xm​)​

Simplifying since we have a X−xm​ in the numerator and denominator:

X−xm​Li​(X)​=A′(xi​)⋅(X−xi​)A′(xm​)⋅Lm​(X)​

This is the exact same formula that we had in Dankrad's post, but generalised! To see this, we note that when the elements in the domain are roots of unity; A′(xk​)=d(xk)d−1=dx−k

The nice simplifcation here is due to two reasons: roots of unity form a cyclic group, and we can succintly represent the d'th roots of unity in a sparse equation Xd−1 which is nice to derivate.

We have now re-defined q(X) to not include X−xm​ !

We now summarise and state that:

q(X)=i=0∑d−1​fi​X−xm​Li​(X)​=fi​A′(xi​)⋅(X−xi​)A′(xm​)⋅Lm​(X)​

when dealing with the point which vanishes on zero, the above formula becomes:

Note: Lm​(xm​)=1

Computing qj​

For the case that the evaluation does not vanish on the domain, we can use the original formula.

For all j=m

qj​=q(xj​)=i=0∑d−1​fi​xj​−xm​Li​(xj​)​

We note that the terms of the sum are zero, except for when i=j, hence we can simplify this to be:

qj​=xj​−xm​fj​​

Optimisations

If we use the formulas as shown above, qm​ will take d steps due to the sum, and qj​ will take d−1 steps. We describe a way to reduce this complexity in the code.

1. Rewrite qm​ in terms of qj​

Note that if we multiply qm​ by −1−1​ we get:

qm​=q(xm​)=−i=0∑d−1​A′(xi​)A′(xm​)​xi​−xm​fi​​

We can now substite in qi​

qm​=q(xm​)=−i=0∑d−1​A′(xi​)A′(xm​)​qi​

2. Removing field inversions in qj​

Note that qj​ has a division which is many times more expensive than a field multiplication. We now show a way to precompute in such a way that we do not need to invert elements.

With the roots of unity, we were able to use the fact that they formed a group.

Again note that:

qj​=xj​−xm​fj​​

(Dankrad):

For our case where the domain is the discrete interval [0,255]

We only need to precompute xi​1​ for xi​∈[−255,255]. This is 510 values, so we would store 510 * 32 = 16Kb.

How would I lookup these values?

First we imagine that we have stored the values in an array:

[11​,21​,31​,41​...2551​,−11​,−21​,...−2551​]

We first note that we can easily get from k1​ to −k1​ in the array by jumping forward 255 indices. Our strategy will be to find k1​ then jump to k1​ if we need to.

Example

We want to compute 0−2551​.

  • Compute the abs(0−255)=255=i

In practice, we can use an if statement to check whether 255 or 0 is larger, and subtract accordingly.

  • Note that i1​ is at index i−1
  • Since our original computation was 0−255 which is negative, we need to get the element at index i−1+255

3. Precompute A′(xi​)A′(xm​)​

With the roots of unity, we did not need this optimisation as A′(xi​)A′(xm​)​ equaled ωmωi​ which was trivial to fetch from the domain.

For this, we will need to store precomputed values, if we want to efficiently compute qm​ in O(d) steps, and to also avoid inversions.

(Dankrad): We precompute A′(xi​) and A′(xi​)1​. Given that we have 256 points in the domain. This will cost us 256 2 32 bytes = 16kB.

How would I lookup these values?

Similar to the previous optimisation, we store A′(xi​) in an array.

[A′(0),A′(1),A′(2),A′(3)...A′(255),A′(0)1​,A′(1)1​,A′(2)1​,...A′(255)1​]

Example

We want to compute A′(5)A′(0)​

  • We can fetch A′(0) by looking up the element at index 0 in the array.
  • We can fetch A′(5)1​ by looking up the element at index 5, then jumping forward 256 positions.

In general:

  • To fetch A(xi​) we need to fetch the element at index i
  • To fetch A(xi​)1​ we need to fetch the element at index i+256

Evaluate polynomial in evaluation form on a point outside of the domain

Suppose z is a point outside of the domain.

  • We already store precomputations for A′(xi​)1​
  • We should compute z−xi​ separately, then batch invert using the montgomery trick, so that we only pay for one inversion.
